## Post-PRD Hardening (beyond the original Ralph Loop)
- [x] Haiku classifier replacing regex silence filters (`message-classifier.ts`)
- [x] Dedicated `phase_confirmation` column; stop reusing `budget_raw` for sub-state flags (migration 030)
- [x] `pending_planners` table dropped as dead code (migration 031)
- [x] Scenario 4 & 6 bulk-run flakes fixed (timeout bumped to 300s)
- [x] `looksLikeDateContent` tightened (month AND day, OR holiday phrase)
- [x] `deno check` in deploy pipeline (`scripts/deploy-sms.sh`)
- [x] Pre-existing TS errors in `polls/[pollId]/edit.tsx` swept
- [x] Asterisk-correction acknowledgment (PRD P5-5 completion): no longer
  silently ignored — bot now responds with "Got it — noted the correction
  to ‹prev snippet›…" when a recent prior message from the same sender
  exists within 2 min. Bare "*…" with no referent still silenced.
- [x] Welcome-back recap (PRD P3-7 completion): when a participant texts
  after >7 days of session silence, Rally prepends a short recap
  (destination, dates, budget, current phase) to whatever response it
  would send. Rate-limited via `trip_sessions.welcome_back_sent_at` so it
  only fires once per gap. Migration 033 adds the column.
- [x] Sentry + PostHog telemetry for Deno edge functions. Lightweight
  `telemetry.ts` uses fetch directly to Sentry's envelope API and
  PostHog's capture endpoint; no-ops when `SENTRY_DSN` or `POSTHOG_KEY`
  env secrets are unset. Call sites: webhook error handler, session
  creation, STOP, graceful opt-out, BOOKED. `supabase/functions/.env.example`
  documents all required secrets.
